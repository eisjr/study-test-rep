# Конечные автоматы (на примере аудио плеера)

## Конечный автомат

В программировании конечный автомат (FSM) — это паттерн проектирования, который помогает управлять сложной логикой переключений. Вместо того чтобы писать сотни запутанных условий if-else или switch, вы четко определяете состояния объекта и правила перехода между ними.Это делает код предсказуемым, его легче тестировать и расширять.

## аудио-плеер

### состояния
| Название | Состояние  |
| :--- | :---: |
| Stopped | воспроизведение остановлено |
| Buffering | идёт буферезация данных |
| Playing | воспроизведение трека |
| Paused | воспроизведение на паузе |

#

### состояния
| Название | События (переходы)  |
| :--- | :---: |
| play | начать или возобновить возпроизведение |
| pause | поставить на паузу |
| stop | остановить воспроизведение |
| buffer | возникла необходимость в буферезации |
| buffer_ready | буферезация завершена, можно воспроизводить |
