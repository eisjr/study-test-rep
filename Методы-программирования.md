

# Методы программирования приложений



#### 1\. Процедурное программирование



Программа = набор процедур/функций

Логика идёт «сверху вниз»

Просто, но плохо масштабируется

Языки: C, Pascal



#### Объектно-ориентированное (ООП)



Основано на классах и объектах

Принципы: инкапсуляция, наследование, полиморфизм

Самый популярный подход для приложений

Языки: Java, C++, C#, Python



#### Функциональное программирование



Всё строится на функциях без состояния

Меньше ошибок, удобно для параллельных вычислений

Языки: Haskell, Scala, Lisp, частично JS, Python



#### Событийно-ориентированное



Приложение реагирует на события (клик, ввод, сигнал)

Основа GUI и мобильных приложений

Примеры: JavaScript, C# (WPF), Android



#### Компонентно-ориентированное



Приложение собирается из независимых компонентов

Удобно для больших систем

Примеры: React, Angular, .NET



#### Клиент-серверное



Логика разделена между клиентом и сервером

Используется в веб- и мобильных приложениях

Примеры: REST API, GraphQL



#### Микросервисная архитектура



Приложение = набор независимых сервисов

Масштабируемо, но сложно

Технологии: Docker, Kubernetes



#### Agile / итеративная разработка (методология)



Не «как писать код», а как разрабатывать приложение

Гибкость, быстрые релизы

Scrum, Kanban
